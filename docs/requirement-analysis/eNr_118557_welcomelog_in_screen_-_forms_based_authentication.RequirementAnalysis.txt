# eNcounter Refresh: Welcome/Log In Screen - Forms Based Authentication Feature Test Plan

## Requirements Analysis

| Scenario Name | Requirement | Test Case |
|---------------|-------------|-----------|
| Scenario 1: Default authentication method (forms based) | 1.1 Welcome screen should display all required components when configured for forms-based authentication | 1.1.1 Verify all welcome screen components display correctly on startup |
|
| Scenario 2: User authenticates successfully (forms based) | 2.1 Valid credentials should trigger navigation to patient intake screen | 2.1.1 Verify successful authentication navigates to patient intake screen |
|
| Scenario 3: User authenticates unsuccessfully (forms based) | 3.1 Invalid credentials should display error with field highlighting and closable message | 3.1.1 Verify failed authentication displays error with red field highlighting and closable "x" |
|
| Scenario 4: User forgot password (forms based) | 4.1 'Forgot your password?' button should navigate to forgot/change password screen | 4.1.1 Verify clicking 'Forgot your password?' navigates to forgot/change password screen |
|
| Scenario 5: Not Licensed (Default to ClearSteth Listen-Only) | 5.1 Application should open as ClearSteth with listen-only functionality when no license is activated | 5.1.1 Verify application opens as ClearSteth listen-only mode with no active license |
|
| Scenario 6: eNcounter License | 6.1 Application should display eNcounter variant of welcome/log in screen with active eNcounter license | 6.1.1 Verify eNcounter variant of welcome screen displays with eNcounter license |
|
| Scenario 7: ClearSteth License | 7.1 Application should display ClearSteth variant of welcome/log in screen with active ClearSteth license | 7.1.1 Verify ClearSteth variant of welcome screen displays with ClearSteth license |
|
| Scenario 8: Ensure Devices are Cleared of Previous Data | 8.1 eNcounter welcome screen should display device clearing message | 8.1.1 Verify device clearing message is displayed on eNcounter welcome screen |
|
| Scenario 9: Sign in with Microsoft | 9.1 eNcounter welcome screen should provide "Sign in With Microsoft" option | 9.1.1 Verify "Sign in With Microsoft" option is available on eNcounter welcome screen |
|
| Scenario 10: Is this an emergency? | 10.1 eNcounter welcome screen should provide "Is this an emergency" option | 10.1.1 Verify "Is this an emergency" option is available on eNcounter welcome screen |
|
| Scenario 11: Copyright Text | 11.1 eNcounter welcome screen should display copyright text at bottom | 11.1.1 Verify copyright text is displayed at bottom of eNcounter welcome screen |

## Detailed Test Cases

| Test Case ID | Test Scenario | Expected Result | Test Data |
|--------------|---------------|-----------------|-----------|
| 1.1.1 | Launch application configured for forms-based authentication and observe all welcome screen components | Welcome screen displays with username field, password field, Sign-In button, 'Forgot your password?' button, Sign in with Microsoft button, device clearing message, 'Is this an Emergency?' button, and copyright text | Application configured for forms-based auth |
| 2.1.1 | Enter valid username and password, then click Sign-In button | Application navigates to patient intake screen | Valid username and password |
| 3.1.1 | Enter invalid username and password, click Sign-In button, observe error display and field highlighting, then click "x" to close | Error message is displayed, username and password fields are highlighted red with alert icon, error message closes when "x" is clicked | Invalid username and password |
| 4.1.1 | Click 'Forgot your password?' button on welcome screen | Application navigates to forgot/change password screen | N/A |
| 5.1.1 | Launch application on host machine with no active license | Application opens as ClearSteth with listen-only functionality | Host machine with no license |
| 6.1.1 | Launch application on host machine with active eNcounter license | eNcounter variant of welcome/log in screen is displayed | Host machine with eNcounter license |
| 7.1.1 | Launch application on host machine with active ClearSteth license | ClearSteth variant of welcome/log in screen is displayed | Host machine with ClearSteth license |
| 8.1.1 | Launch eNcounter application and observe welcome screen | Message "Please ensure you have cleared all devices of previous patient data." is displayed | eNcounter license active |
| 9.1.1 | Launch eNcounter application and observe welcome screen | "Sign in With Microsoft" option is available | eNcounter license active |
| 10.1.1 | Launch eNcounter application and observe welcome screen | "Is this an emergency" option is available | eNcounter license active |
| 11.1.1 | Launch eNcounter application and observe welcome screen | Copyright text is displayed at bottom of screen | eNcounter license active |
